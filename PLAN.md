# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature 8900ab21)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest, leave application workflow, approval and rejection state transitions
- **balance** (`src/modules/balance`) — owns: LeaveBalance, leave balance adjustments, leave balance queries
- **employee** (`src/modules/employee`) — owns: Employee, manager relationships, employee status data
- **policy** (`src/modules/policy`) — owns: LeavePolicy, leave entitlement rules, leave type validation rules
- **notification** (`src/modules/notification`) — owns: Notification, leave workflow notifications, notification delivery records

## Domain entities
- **LeaveRequest** — Represents an employee leave application and its approval workflow state. (attributes: id, employeeId, leaveType, startDate, endDate, status)
- **LeaveBalance** — Tracks remaining leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, availableDays)
- **Employee** — Represents employee identity and reporting hierarchy used in leave approvals. (attributes: id, managerId, status)
- **LeavePolicy** — Defines leave entitlement and validation rules by leave type. (attributes: id, leaveType, annualEntitlement)
- **Notification** — Represents notifications generated during leave request processing. (attributes: id, recipientEmployeeId, type, status)

# PLAN.md

## Phase 1: Phase 1: Shared types and base repository

Create src/shared/types/index.ts with EmploymentStatus enum (Active, Inactive, Terminated, OnLeave), LeaveStatus enum (Pending, Approved, Rejected, Cancelled), and BaseEntity interface (id: string, createdAt: Date, updatedAt: Date, deletedAt: Date | null). Create src/shared/base.repository.ts with an abstract BaseRepository<T> class providing common CRUD methods (findById, findAll, create, update, softDelete) using the pg Pool from src/shared/db/connection.ts. Include Jest unit tests in tests/unit/shared/.

## Phase 2: Phase 2: Employee module — model and repository

Create src/modules/employee/employee.model.ts with the Employee interface (id: string, employeeNumber: string, firstName: string, lastName: string, email: string, managerId: string | null, department: string | null, hireDate: Date, terminationDate: Date | null, employmentStatus: EmploymentStatus, createdAt: Date, updatedAt: Date, deletedAt: Date | null) extending BaseEntity from src/shared/types/index.ts. Also define CreateEmployeeDto and UpdateEmployeeDto. Create src/modules/employee/employee.repository.ts implementing IEmployeeRepository that extends BaseRepository<Employee> from src/shared/base.repository.ts, adding findByEmployeeNumber and findByManagerId. Create src/modules/employee/index.ts as the public entry point. Include Jest unit tests in tests/unit/modules/employee/.

## Phase 3: Phase 3: LeaveType and LeavePolicy — models and repositories

Create src/modules/leaveType/leaveType.model.ts with LeaveType interface (code: string, label: string, description: string | null, isActive: boolean). Create src/modules/leavePolicy/leavePolicy.model.ts with LeavePolicy interface (id: string, policyName: string, leaveTypeCode: string, entitlementDays: number, accrualRate: number | null, maxAccumulation: number | null, minimumNoticeDays: number | null, requiresManagerApproval: boolean, isActive: boolean, createdAt: Date, updatedAt: Date) extending BaseEntity from src/shared/types/index.ts. Also define CreateLeavePolicyDto and UpdateLeavePolicyDto. Create src/modules/leaveType/leaveType.repository.ts and src/modules/leavePolicy/leavePolicy.repository.ts, each extending BaseRepository. Create index.ts barrel files for both modules. Include Jest unit tests in tests/unit/modules/leaveType/ and tests/unit/modules/leavePolicy/.

## Phase 4: Phase 4: LeaveBalance — model and repository

Create src/modules/leaveBalance/leaveBalance.model.ts with LeaveBalance interface (id: string, employeeId: string, leaveTypeCode: string, policyId: string, entitlementDays: number, usedDays: number, pendingDays: number, accruedDays: number, carriedForwardDays: number, year: number, createdAt: Date, updatedAt: Date) extending BaseEntity from src/shared/types/index.ts. Also define CreateLeaveBalanceDto and UpdateLeaveBalanceDto. Create src/modules/leaveBalance/leaveBalance.repository.ts implementing ILeaveBalanceRepository that extends BaseRepository<LeaveBalance>, adding findByEmployeeIdAndYear, findByEmployeeIdAndLeaveTypeCode, and upsert methods. This phase depends on src/modules/employee/employee.model.ts from Phase 2 and src/modules/leaveType/leaveType.model.ts and src/modules/leavePolicy/leavePolicy.model.ts from Phase 3 — read them before generating any code. Create src/modules/leaveBalance/index.ts. Include Jest unit tests in tests/unit/modules/leaveBalance/.

## Phase 5: Phase 5: LeaveRequest — model and repository

Create src/modules/leaveRequest/leaveRequest.model.ts with LeaveRequest interface (id: string, employeeId: string, leaveTypeCode: string, policyId: string, startDate: Date, endDate: Date, totalDays: number, status: LeaveStatus, reason: string | null, reviewerId: string | null, reviewedAt: Date | null, rejectionReason: string | null, createdAt: Date, updatedAt: Date) extending BaseEntity from src/shared/types/index.ts. Also define CreateLeaveRequestDto and UpdateLeaveRequestDto. Create src/modules/leaveRequest/leaveRequest.repository.ts implementing ILeaveRequestRepository that extends BaseRepository<LeaveRequest>, adding findByEmployeeId, findByStatus, findByDateRange, and findPendingByManagerId. This phase depends on src/modules/employee/employee.model.ts from Phase 2, src/modules/leaveType/leaveType.model.ts and src/modules/leavePolicy/leavePolicy.model.ts from Phase 3, and src/modules/leaveBalance/leaveBalance.model.ts from Phase 4 — read them before generating any code. Create src/modules/leaveRequest/index.ts. Include Jest unit tests in tests/unit/modules/leaveRequest/.

## Phase 6: Phase 6: LeaveRequest service — business logic and validation

Create src/modules/leaveRequest/leaveRequest.service.interface.ts with ILeaveRequestService interface (apply, approve, reject, cancel, getById, getByEmployee, getPendingForManager). Create src/modules/leaveRequest/leaveRequest.service.ts implementing ILeaveRequestService with full business logic: validate leave policy rules (minimum notice, manager approval requirement), check for overlapping requests, validate sufficient balance via ILeaveBalanceRepository, enforce LeaveStatus state machine transitions (Pending→Approved/Rejected/Cancelled, Approved→Cancelled), and update LeaveBalance pendingDays/usedDays on status changes. This phase depends on src/modules/leaveRequest/leaveRequest.model.ts and src/modules/leaveRequest/leaveRequest.repository.ts from Phase 5, src/modules/leaveBalance/leaveBalance.repository.ts from Phase 4, src/modules/leavePolicy/leavePolicy.repository.ts from Phase 3, and src/modules/employee/employee.repository.ts from Phase 2 — read them before generating any code. Update src/modules/leaveRequest/index.ts to export the new service. Include Jest unit tests in tests/unit/modules/leaveRequest/.

## Phase 9: Phase 9: LeaveRequest routes and controller with RBAC

Create src/modules/leaveRequest/leaveRequest.controller.ts with LeaveRequestController class that delegates to ILeaveRequestService and handles HTTP request/response mapping. Create src/modules/leaveRequest/leaveRequest.routes.ts registering Fastify routes: POST /leave-requests (apply), PATCH /leave-requests/:id/approve, PATCH /leave-requests/:id/reject, PATCH /leave-requests/:id/cancel, GET /leave-requests/:id, GET /leave-requests (employee's own), GET /leave-requests/pending (manager's pending). Implement RBAC middleware: employee role can apply/cancel own requests; manager role can approve/reject reports' requests; HR admin can do all. Wire AuditLog and Notification service calls into the controller so every state change is audited and notified. This phase depends on src/modules/leaveRequest/leaveRequest.service.ts from Phase 6, src/modules/auditLog/auditLog.service.ts from Phase 7, and src/modules/notification/notification.service.ts from Phase 8 — read them before generating any code. Update src/modules/leaveRequest/index.ts. Register routes in src/app.ts. Include Jest integration tests in tests/integration/modules/leaveRequest/.

## Phase 10: Phase 10: LeaveBalance service — accrual and recalculation logic

Create src/modules/leaveBalance/leaveBalance.service.interface.ts with ILeaveBalanceService interface (getBalance, recalculate, accrue, initializeForEmployee). Create src/modules/leaveBalance/leaveBalance.service.ts implementing ILeaveBalanceService: recalculate method recomputes usedDays and pendingDays from LeaveRequest repository; accrue method applies accrualRate from LeavePolicy to accruedDays respecting maxAccumulation; initializeForEmployee creates balance records for all active policies when a new employee is hired. This phase depends on src/modules/leaveBalance/leaveBalance.model.ts and src/modules/leaveBalance/leaveBalance.repository.ts from Phase 4, src/modules/leavePolicy/leavePolicy.repository.ts from Phase 3, and src/modules/leaveRequest/leaveRequest.repository.ts from Phase 5 — read them before generating any code. Update src/modules/leaveBalance/index.ts. Include Jest unit tests in tests/unit/modules/leaveBalance/.

# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature a57e62c3)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest entity lifecycle, Leave approval workflow, Leave request validation
- **balance** (`src/modules/balance`) — owns: LeaveBalance entity, Balance adjustments, Balance queries
- **employee** (`src/modules/employee`) — owns: Employee entity, Manager-employee relationships, Employee status
- **policy** (`src/modules/policy`) — owns: LeavePolicy entity, Policy rules, Entitlement calculations
- **notification** (`src/modules/notification`) — owns: Notification entity, Notification delivery, Notification templates

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee, processed through approval workflow (attributes: id, employeeId, leaveType, startDate, endDate, status, reason, managerId, createdAt, updatedAt)
- **LeaveBalance** — Tracks remaining leave entitlement for an employee by leave type and fiscal year (attributes: id, employeeId, leaveType, balance, fiscalYear)
- **Employee** — Employee identity and reporting hierarchy for leave approval routing (attributes: id, name, email, managerId, department, isActive)
- **LeavePolicy** — Defines entitlement rules and constraints for each leave type (attributes: id, leaveType, entitlementDays, carryOverLimit, requiresApproval)
- **Notification** — Leave workflow notifications for employees and managers (attributes: id, recipientId, type, title, body, metadata, status, createdAt)

# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature d0513f28)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest lifecycle, Leave application submission, Approval and rejection state transitions, Workflow orchestration
- **balance** (`src/modules/balance`) — owns: LeaveBalance records, Balance adjustments, Balance retrieval
- **employee** (`src/modules/employee`) — owns: Employee records, Manager relationships
- **policy** (`src/modules/policy`) — owns: Leave entitlement rules, Leave type policies
- **notification** (`src/modules/notification`) — owns: Notification creation, Notification delivery status

## Domain entities
- **LeaveRequest** — Represents a leave application and its approval workflow state. (attributes: id, employeeId, leaveType, startDate, endDate, status, approverEmployeeId, createdAt, updatedAt)
- **LeaveBalance** — Tracks available leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, remainingDays)
- **Employee** — Represents an employee and reporting hierarchy used for approvals. (attributes: id, managerId, name)
- **LeavePolicy** — Defines leave eligibility and entitlement rules. (attributes: id, leaveType, annualEntitlement)
- **Notification** — Represents workflow notifications generated by leave events. (attributes: id, recipientEmployeeId, type, status, relatedLeaveRequestId, createdAt)

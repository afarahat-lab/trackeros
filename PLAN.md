# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature e1ee9e5c)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest lifecycle, Leave approval workflow, Leave application validation
- **balance** (`src/modules/balance`) — owns: LeaveBalance storage, Balance adjustments, Balance queries
- **employee** (`src/modules/employee`) — owns: Employee identity, Reporting hierarchy, Manager relationships
- **policy** (`src/modules/policy`) — owns: LeavePolicy rules, Entitlement calculations, Leave type definitions
- **notification** (`src/modules/notification`) — owns: Notification delivery, Notification templates, Notification preferences

## Domain entities
- **Employee** — Employee identity and reporting hierarchy (attributes: id, employeeNumber, firstName, lastName, email, managerId, department, hireDate, isActive)
- **LeavePolicy** — Entitlement and leave-type rules (attributes: id, policyName, leaveType, entitlementDays, accrualRate, maxCarryover, requiresApproval, isActive)
- **LeaveRequest** — Leave applications and approval workflow (attributes: id, employeeId, leavePolicyId, startDate, endDate, totalDays, status, reason, managerNotes, approvedBy, approvedAt)
- **LeaveBalance** — Remaining entitlement by employee and leave type (attributes: id, employeeId, leavePolicyId, year, accruedDays, usedDays, carriedOverDays)
- **Notification** — Leave workflow notifications (attributes: id, recipientId, type, title, message, metadata, isRead, readAt)
- **AuditLog** — Audit trail for state-changing operations (attributes: id, tableName, recordId, operation, oldValues, newValues, performedBy, performedAt)

# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature b58ee152)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **Leave** (`src/modules/leave`) — owns: LeaveRequest lifecycle, leave application, approval decisions, request status tracking
- **Balance** (`src/modules/balance`) — owns: LeaveBalance records, balance calculations, balance adjustments
- **Employee** (`src/modules/employee`) — owns: Employee data, manager relationships
- **Policy** (`src/modules/policy`) — owns: Leave entitlement policies, supported leave types
- **Notification** (`src/modules/notification`) — owns: Leave-related notifications, notification delivery state

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee and processed by a manager. (attributes: id, employeeId, leaveType, status, startDate, endDate)
- **LeaveBalance** — Tracks remaining leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, availableDays)
- **Employee** — Represents employees and reporting relationships used during leave approval. (attributes: id, managerId, name)
- **LeavePolicy** — Defines leave entitlement rules and supported leave types. (attributes: id, leaveType, annualAllocation)
- **Notification** — Represents approval and rejection notifications related to leave requests. (attributes: id, recipientEmployeeId, type, status)

# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature 530d359e)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest lifecycle, leave approval workflow, leave request validation
- **balance** (`src/modules/balance`) — owns: LeaveBalance storage, balance adjustments, balance calculations
- **employee** (`src/modules/employee`) — owns: Employee data, reporting hierarchy, employee lookup
- **policy** (`src/modules/policy`) — owns: LeavePolicy rules, leave type definitions, entitlement configuration
- **notification** (`src/modules/notification`) — owns: Notification creation, notification delivery, notification status tracking

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee, processed by managers (attributes: id, employeeId, leaveType, startDate, endDate, status, reason, managerId, createdAt, updatedAt)
- **LeaveBalance** — Tracks remaining leave entitlement by employee and leave type (attributes: id, employeeId, leaveType, balance, fiscalYear)
- **Employee** — Employee identity and reporting hierarchy (attributes: id, name, email, managerId, department, hireDate)
- **LeavePolicy** — Defines entitlement and leave-type rules (attributes: id, leaveType, entitlementDays, carryOverLimit, requiresApproval)
- **Notification** — Leave workflow notifications sent to employees and managers (attributes: id, recipientId, type, title, message, read, createdAt)

# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature 61953f63)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: leave request creation, leave request approval and rejection workflow, leave request status transitions
- **balance** (`src/modules/balance`) — owns: leave balance records, leave balance adjustments, leave availability tracking
- **employee** (`src/modules/employee`) — owns: employee records, manager relationships
- **policy** (`src/modules/policy`) — owns: leave entitlement rules, leave type policies
- **notification** (`src/modules/notification`) — owns: leave workflow notifications, notification delivery records

## Domain entities
- **LeaveRequest** — Represents a leave application and its approval lifecycle. (attributes: id, employeeId, leaveType, startDate, endDate, status, approverEmployeeId, createdAt)
- **LeaveBalance** — Tracks remaining leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, availableDays)
- **Employee** — Represents employees and reporting relationships used in leave approvals. (attributes: id, managerId, name)
- **LeavePolicy** — Defines leave rules and entitlement limits. (attributes: id, leaveType, annualEntitlement)
- **Notification** — Represents workflow notifications generated by leave events. (attributes: id, recipientEmployeeId, type, status)

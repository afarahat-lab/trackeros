# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature ce9d1b80)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: leave request lifecycle, approval workflow, request status transitions
- **balance** (`src/modules/balance`) — owns: leave balance records, balance adjustments, leave entitlement consumption
- **employee** (`src/modules/employee`) — owns: employee records, manager relationships
- **policy** (`src/modules/policy`) — owns: leave policy definitions, entitlement rules
- **notification** (`src/modules/notification`) — owns: notification creation, notification delivery tracking

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee and processed through approval workflows. (attributes: id, employeeId, leaveType, startDate, endDate, status)
- **LeaveBalance** — Tracks remaining leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, availableDays)
- **Employee** — Represents employees and reporting relationships used for leave approval routing. (attributes: id, managerId, name)
- **LeavePolicy** — Defines leave eligibility and entitlement rules. (attributes: id, leaveType, entitlementDays)
- **Notification** — Represents workflow notifications generated by leave request events. (attributes: id, recipientId, type, status)

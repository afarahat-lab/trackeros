# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature 523e9824)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest lifecycle, approval workflows, leave application processing
- **balance** (`src/modules/balance`) — owns: LeaveBalance storage, balance adjustments, balance queries
- **employee** (`src/modules/employee`) — owns: Employee records, manager relationships, department assignments
- **policy** (`src/modules/policy`) — owns: LeavePolicy rules, entitlement configurations
- **notification** (`src/modules/notification`) — owns: Notification creation, delivery, and read status tracking

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee, processed by managers (attributes: id, employeeId, leaveType, startDate, endDate, status, reason, managerId, createdAt, updatedAt)
- **LeaveBalance** — Tracks remaining leave entitlement by employee and leave type (attributes: id, employeeId, leaveType, balance, fiscalYear)
- **Employee** — Employee identity and reporting hierarchy for leave approvals (attributes: id, name, email, managerId, department, hireDate)
- **LeavePolicy** — Defines entitlement rules and leave-type configurations (attributes: id, leaveType, entitlementDays, carryOverLimit, requiresApproval)
- **Notification** — Leave workflow notifications for employees and managers (attributes: id, recipientId, type, title, message, metadata, read, createdAt)

# PLAN.md — Leave management module

_Generated by Gestalt planning layer (feature 1a5dcfc5)._

## Feature description

Build the leave management module. Employees can apply for annual, sick, and emergency leave. Managers approve or reject requests. The system enforces leave balance limits and prevents overlapping requests.

## Modules
- **leave** (`src/modules/leave`) — owns: leave.model.ts, leave.repository.ts, leave.service.ts, leave.routes.ts, leave.test.ts
- **balance** (`src/modules/balance`) — owns: balance.model.ts, balance.repository.ts, balance.service.ts
- **policy** (`src/modules/policy`) — owns: policy.model.ts, policy.repository.ts, policy.service.ts, policy.routes.ts
- **employee** (`src/modules/employee`) — owns: employee.model.ts, employee.repository.ts, employee.service.ts, employee.routes.ts

## Domain entities
- **LeaveRequest** — Represents a leave request made by an employee, including details about the type of leave, duration, and approval status. (attributes: id, employeeId, type, startDate, endDate, status, managerId, managerComment, createdAt)
- **LeaveBalance** — Tracks the leave balance for each employee, ensuring that leave requests can be validated against available leave. (attributes: employeeId, leaveType, totalDays, usedDays, year)
- **LeavePolicy** — Defines the rules and limits for different types of leave, which HR can configure. (attributes: id, leaveType, defaultDaysPerYear, maxConsecutiveDays, requiresApproval, createdAt)

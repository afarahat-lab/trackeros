# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature 48aa490e)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest, leave application workflow, approval, rejection, and cancellation state transitions, audit events for leave workflow
- **balance** (`src/modules/balance`) — owns: LeaveBalance, leave balance adjustments, leave entitlement consumption and restoration
- **employee** (`src/modules/employee`) — owns: Employee, manager relationships, employee role information
- **policy** (`src/modules/policy`) — owns: LeavePolicy, leave eligibility and entitlement rules
- **notification** (`src/modules/notification`) — owns: Notification, leave workflow notifications
- **audit** (`src/modules/audit`) — owns: AuditRecord, audit record persistence for all state-changing operations

## Domain entities
- **LeaveRequest** — Represents a leave application and its approval workflow state. (attributes: id, employeeId, leaveType, startDate, endDate, status)
- **LeaveBalance** — Tracks available leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, remainingDays)
- **Employee** — Represents employees and reporting relationships used for approvals and RBAC. (attributes: id, managerId, role)
- **LeavePolicy** — Defines leave rules and entitlement limits. (attributes: id, leaveType, annualEntitlement)
- **Notification** — Represents workflow notifications generated by leave actions. (attributes: id, recipientEmployeeId, type, status)
- **AuditRecord** — Records state-changing operations for compliance and traceability. (attributes: id, entityType, entityId, action, createdAt)

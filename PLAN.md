# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature fc99779a)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest, leave application workflow, approval and rejection state transitions, leave-related audit events
- **balance** (`src/modules/balance`) — owns: LeaveBalance, leave balance adjustments and retrieval
- **employee** (`src/modules/employee`) — owns: Employee, manager relationships, employee role information
- **policy** (`src/modules/policy`) — owns: LeavePolicy, leave entitlement rules and validation data
- **notification** (`src/modules/notification`) — owns: Notification, leave workflow notifications
- **audit** (`src/modules/audit`) — owns: AuditRecord, audit persistence for state-changing operations

## Domain entities
- **LeaveRequest** — Represents a leave application and its approval lifecycle. (attributes: id, employeeId, leaveType, startDate, endDate, status)
- **LeaveBalance** — Tracks remaining leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, availableDays)
- **Employee** — Represents employees and reporting relationships used in leave approval workflows. (attributes: id, managerId, role)
- **LeavePolicy** — Defines leave rules and entitlement limits. (attributes: id, leaveType, annualEntitlement)
- **Notification** — Represents workflow notifications generated by leave events. (attributes: id, recipientEmployeeId, type, status)
- **AuditRecord** — Captures state-changing leave operations for auditability. (attributes: id, entityType, entityId, action, createdAt)

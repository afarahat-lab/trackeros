# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature d90d14b5)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest, leave request submission, leave approval workflow, leave rejection workflow, audit coordination for leave state changes
- **balance** (`src/modules/balance`) — owns: LeaveBalance, leave balance adjustments, leave balance queries
- **employee** (`src/modules/employee`) — owns: Employee, manager relationships, employee role data
- **policy** (`src/modules/policy`) — owns: LeavePolicy, leave entitlement rules, leave type validation rules
- **notification** (`src/modules/notification`) — owns: Notification, leave workflow notifications
- **audit** (`src/modules/audit`) — owns: AuditRecord, audit record persistence

## Domain entities
- **LeaveRequest** — Represents a leave application and its approval workflow. (attributes: id, employeeId, leaveType, startDate, endDate, status)
- **LeaveBalance** — Tracks available leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, remainingDays)
- **Employee** — Represents employees and reporting relationships used for leave approvals and RBAC. (attributes: id, managerId, role)
- **LeavePolicy** — Defines leave entitlement and validation rules for leave types. (attributes: id, leaveType, annualEntitlement)
- **Notification** — Represents notifications generated during leave request processing. (attributes: id, recipientEmployeeId, type, status)
- **AuditRecord** — Stores audit history for all leave-related state changes. (attributes: id, entityType, entityId, action, actorEmployeeId, createdAt)

# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature dca0cb06)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest, leave request submission, manager approval workflow, manager rejection workflow
- **balance** (`src/modules/balance`) — owns: LeaveBalance, leave balance adjustments
- **employee** (`src/modules/employee`) — owns: Employee, manager relationships, employee role data
- **policy** (`src/modules/policy`) — owns: LeavePolicy, leave entitlement rules, supported leave type definitions
- **notification** (`src/modules/notification`) — owns: Notification, leave workflow notifications
- **audit** (`src/modules/audit`) — owns: AuditRecord, audit history for state-changing operations

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee and processed through manager approval workflows. (attributes: id, employeeId, leaveType, status)
- **LeaveBalance** — Tracks available leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, remainingDays)
- **Employee** — Represents employees and reporting relationships used for leave approval routing and RBAC. (attributes: id, managerId, role)
- **LeavePolicy** — Defines leave entitlement rules for supported leave types. (attributes: id, leaveType, annualEntitlement)
- **Notification** — Represents workflow notifications generated by leave request state changes. (attributes: id, recipientEmployeeId, type, status)
- **AuditRecord** — Stores mandatory audit history for all leave-related state-changing operations. (attributes: id, entityType, entityId, action)

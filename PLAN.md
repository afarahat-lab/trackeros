# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature 54a891da)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest entity, LeaveRequest repository, LeaveRequest service, LeaveRequest controller, LeaveRequest routes, Leave approval workflow
- **balance** (`src/modules/balance`) — owns: LeaveBalance entity, LeaveBalance repository, LeaveBalance service, LeaveBalance controller, LeaveBalance routes, Balance adjustment logic
- **employee** (`src/modules/employee`) — owns: Employee entity, Employee repository, Employee service, Employee controller, Employee routes, Reporting hierarchy
- **policy** (`src/modules/policy`) — owns: LeavePolicy entity, LeavePolicy repository, LeavePolicy service, LeavePolicy controller, LeavePolicy routes, Entitlement rules
- **notification** (`src/modules/notification`) — owns: Notification entity, Notification repository, Notification service, Notification controller, Notification routes, Leave-related alerts

## Domain entities
- **LeaveRequest** — Captures an employee's leave application and its approval lifecycle. (attributes: id, employeeId, leaveType, startDate, endDate, status, reason, requestedAt, reviewedBy, reviewedAt, reviewNote)
- **LeaveBalance** — Tracks remaining leave entitlement per employee and leave type. (attributes: id, employeeId, leaveType, totalDays, usedDays, remainingDays, year)
- **Employee** — Employee identity and reporting hierarchy for routing approvals. (attributes: id, name, email, managerId)
- **LeavePolicy** — Defines entitlement rules per leave type. (attributes: id, leaveType, entitlementDays, carryOverAllowed, year)
- **Notification** — Alerts stakeholders about leave workflow events. (attributes: id, userId, type, message, relatedEntityId, isRead, createdAt)

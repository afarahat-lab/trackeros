# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature e3298836)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest entity lifecycle, Leave approval workflow, Leave request validation
- **balance** (`src/modules/balance`) — owns: LeaveBalance entity storage, Balance adjustments, Balance queries
- **employee** (`src/modules/employee`) — owns: Employee entity, Reporting hierarchy, Employee queries
- **policy** (`src/modules/policy`) — owns: LeavePolicy entity, Policy rules, Entitlement definitions
- **notification** (`src/modules/notification`) — owns: Notification entity, Notification delivery, Notification templates

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee, tracking its lifecycle from submission through approval/rejection. (attributes: id, employeeId, leaveType, startDate, endDate, status, reason, managerId, createdAt, updatedAt)
- **LeaveBalance** — Tracks remaining leave entitlement for an employee by leave type and fiscal year. (attributes: id, employeeId, leaveType, balance, fiscalYear)
- **Employee** — Represents an employee identity and their reporting hierarchy (manager relationship). (attributes: id, name, email, managerId, department)
- **LeavePolicy** — Defines rules and entitlements for each type of leave (annual, sick, emergency). (attributes: id, leaveType, entitlementDays, carryOverLimit, requiresApproval)
- **Notification** — Represents a notification sent to users about leave workflow events (submission, approval, rejection). (attributes: id, userId, type, title, message, read, createdAt)

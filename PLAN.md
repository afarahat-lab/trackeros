# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature ae9bd00b)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest lifecycle, approval workflows, leave application validation
- **balance** (`src/modules/balance`) — owns: LeaveBalance storage, balance adjustments, balance queries
- **employee** (`src/modules/employee`) — owns: Employee identity, manager relationships, department hierarchy
- **policy** (`src/modules/policy`) — owns: LeavePolicy rules, entitlement definitions, leave type configurations
- **notification** (`src/modules/notification`) — owns: Notification creation, delivery, status tracking

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee and processed by a manager (attributes: id, employeeId, leaveType, startDate, endDate, status, reason, managerId, createdAt, updatedAt)
- **LeaveBalance** — Tracks remaining leave entitlement by employee and leave type (attributes: id, employeeId, leaveType, balance, fiscalYear)
- **Employee** — Employee identity and reporting hierarchy for leave approval workflow (attributes: id, name, email, managerId, department, employmentDate)
- **LeavePolicy** — Defines entitlement rules and leave-type configurations (attributes: id, leaveType, entitlementDays, carryOverLimit, requiresApproval)
- **Notification** — Leave workflow notifications for employees and managers (attributes: id, recipientId, type, title, body, metadata, status, createdAt)

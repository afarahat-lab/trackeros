# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature 795e1069)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest, leave request lifecycle, approval and rejection workflow
- **balance** (`src/modules/balance`) — owns: LeaveBalance, leave balance adjustments
- **employee** (`src/modules/employee`) — owns: Employee, manager relationships, employee role information
- **policy** (`src/modules/policy`) — owns: LeavePolicy, leave entitlement rules
- **notification** (`src/modules/notification`) — owns: Notification, leave workflow notifications
- **audit** (`src/modules/audit`) — owns: AuditRecord, audit record persistence

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee and processed through approval workflow. (attributes: id, employeeId, leaveType, status)
- **LeaveBalance** — Tracks available leave entitlement for each employee and leave type. (attributes: employeeId, leaveType, remainingDays)
- **Employee** — Represents employees and reporting relationships used for leave approval and RBAC. (attributes: id, managerId, role)
- **LeavePolicy** — Defines leave entitlement and eligibility rules. (attributes: id, leaveType, annualEntitlement)
- **Notification** — Represents workflow notifications generated by leave events. (attributes: id, recipientEmployeeId, type, status)
- **AuditRecord** — Captures audit history for all leave workflow state-changing operations. (attributes: id, entityType, entityId, action)

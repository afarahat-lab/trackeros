# PLAN.md — Build the leave management module

_Generated by Gestalt planning layer (feature 1a6a0bc1)._

## Feature description

Build the leave management module. Employees apply for annual, sick, and emergency leave. Managers approve or reject. System tracks leave balances.

## Modules
- **leave** (`src/modules/leave`) — owns: LeaveRequest lifecycle, approval workflow, leave application processing
- **balance** (`src/modules/balance`) — owns: LeaveBalance storage, balance adjustments, balance queries
- **employee** (`src/modules/employee`) — owns: Employee data, manager relationships, employee queries
- **policy** (`src/modules/policy`) — owns: LeavePolicy rules, policy validation, policy queries
- **notification** (`src/modules/notification`) — owns: Notification creation, delivery, and status tracking

## Domain entities
- **LeaveRequest** — Represents a leave application submitted by an employee, processed through approval workflow (attributes: id, employeeId, leaveType, startDate, endDate, status, reason, managerId, createdAt, updatedAt)
- **LeaveBalance** — Tracks remaining leave entitlement for each employee by leave type and fiscal year (attributes: id, employeeId, leaveType, balance, fiscalYear)
- **Employee** — Employee identity and reporting hierarchy for leave request routing (attributes: id, name, email, managerId, department)
- **LeavePolicy** — Defines rules for leave types including entitlement and approval requirements (attributes: id, leaveType, entitlementDays, carryOverLimit, requiresApproval)
- **Notification** — Leave workflow notifications for employees and managers (attributes: id, recipientId, type, title, message, read, createdAt)
